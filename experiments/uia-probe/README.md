# UiaProbe (検証用・使い捨て)

Claude Desktop のウィンドウが、Windows UI Automation で外部から読み取れるかどうかを
検証するために作った使い捨てのコンソールアプリです。**claude-signal-tray 本体には
組み込まれていません。**

## 検証結果（結論: 断念）

- 通常表示時・最小化時のどちらでも、UI Automationで取得できたのは
  「最小化」「最大化」「閉じる」ボタンなど、OSのウィンドウ枠（12要素）のみでした。
- チャット内容や確認ボタンなど、Claude Desktop（Electron/Chromium製と思われる）が
  描画している実際のUI内容は一切露出しませんでした。
- 最小化の有無で結果に差は無く、「最小化で壊れる」のではなく「そもそも
  Chromiumの中身がUI Automationクライアントに公開されていない」ことが根本原因と
  判断しました。

Electron/Chromium系アプリは、既定では外部のUI Automationクライアントにアクセシビ
リティツリーを公開しません。スクリーンリーダー（ナレーターなど）が動作している
ことを検知した場合にのみ開放する実装が一般的です。この抜け道はありますが、
「常にナレーターを起動しておく」という不自然な前提が必要になるため、
claude-signal-tray では採用しませんでした。最終的に、当時採用していた
status.json（Coworkのグローバル指示経由）方式も別の理由で不採用となり、現在は
プロセスI/O量ベースの方式に置き換わっています。詳細は
[docs/HISTORY.md](../../docs/HISTORY.md) を参照してください。

## 実行方法（興味があれば）

```
dotnet run
```

Claude Desktopを開いた状態で実行し、指示に従って最小化してEnterを押すと、
最小化前後でのUI Automationツリーの読み取り結果を比較表示します。
