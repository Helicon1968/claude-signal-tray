# Claude Cowork グローバル指示への貼り付け文（v1・非推奨・過去の記録）

> **このv1方式(status.jsonへの書き込みをClaudeに指示する方式)は廃止されました。**
> 実機検証で、指示文をどれだけ強く・明確に書いても、Claudeが実行しない
> ケースが繰り返し確認されたためです。現在の claude-signal-tray は
> [README](../README.md) に記載のネットワークI/O方式（v2）を使用しています。
> 本ページは経緯の記録として残しています。

このツールが機能するには、Claude Desktop の `設定 > Cowork > グローバル指示` に、
以下の指示文を貼り付けて保存する必要があります。

> **注意:** これはClaude Cowork公式のステータスAPIではありません。Claudeが
> 指示に従って正しくファイルを更新し続けることに依存した非公式の仕組みです。
> 実際に検証したところ、「作業開始・終了のたびに書く」という曖昧な指示は、
> 単純な質問応答タスクではClaudeに無視されやすいことが分かりました。そのため
> 以下の指示文は、**「確認待ち(status=2)への切り替えだけは絶対に守る」**ことを
> 最優先にし、0/1の切り替えは努力目標に格下げしています。これは、そもそも
> このツールを作った動機が「確認待ちに気づかない」ことの解消だったためです。

---

**貼り付ける文章（ここから）**

【最優先・例外なし】AskUserQuestion、権限承認、ファイル削除確認など、
ユーザーの入力や確認を待つ操作を行う直前には、必ず先に
`%LOCALAPPDATA%\ClaudeSignalTray\status.json` を次の内容に書き換えてから
確認を出すこと。これは些細な質問やタスクであっても例外なく行う。

```json
{ "status": 2, "task": "確認内容の短い説明", "updated_at": "現在時刻のISO8601形式" }
```

ユーザーが応答して作業を再開したら、同じファイルを次のように書き換える。

```json
{ "status": 1, "task": "作業内容の短い説明", "updated_at": "現在時刻のISO8601形式" }
```

（余裕があれば・必須ではない）タスクを開始し作業を始めた時点で status を 1 に、
タスクが完了しユーザーへの応答を終えて次の入力を待つだけになった時点で status を 0
にできると理想的だが、これは努力目標であり、status=2への切り替えの徹底を最優先とする。

**貼り付ける文章（ここまで）**

---

`%LOCALAPPDATA%` はユーザーによって実パスが異なります（例:
`C:\Users\<ユーザー名>\AppData\Local\ClaudeSignalTray\status.json`）。
Claudeは通常この環境変数を自動的に解決できますが、うまくいかない場合は
実パスに置き換えて指示してください。

## 既知の限界（検証結果より）

新しいCoworkタスクで単純な質問（Web検索して回答するだけなど）を試したところ、
「作業開始/終了時に書く」という曖昧な指示は実行されず、status.jsonへの書き込みは
一度も行われませんでした。

その後、この文書の指示文を「AskUserQuestionの直前は必ず書く」という、より明確で
限定的なトリガーに書き換えて再検証しましたが、実際にAskUserQuestionを使う
タスク（PowerPointのフォントサイズ調整の確認）で試したところ、それでも
status.jsonへの書き込みは一度も行われませんでした。つまり、指示文をどれだけ
明確・強力に書いても、Claudeが確実に実行する保証は無いという結論に至っています。

代替として、Claude DesktopのウィンドウをWindows UI Automationで外部から監視する
方式も検証しましたが、実際のUI内容が外部クライアントに公開されておらず断念しました
（[experiments/uia-probe](../experiments/uia-probe) 参照）。

現時点でこれ以上の確実性を持たせる方法が無いため、**本ツールは「うまく動けば便利」
程度の補助機能である**という前提で使ってください（更新が3分止まると自動的に
「不明(灰)」表示になるのはこのためです）。
